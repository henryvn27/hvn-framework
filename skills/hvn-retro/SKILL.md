# HVN Retro

## What This Skill Is

A learning workflow for improving future HVN work.

## Trigger

Use after release, incident, major feature, or substantial agent workflow.

## Do Not Trigger

Do not use before the work has an outcome to examine.

## Required Inputs

- Completed work summary

## Optional Inputs

- Spec
- Plan
- Review report
- QA reports
- Release notes

## Exact Workflow

1. Compare outcome to the original spec.
2. Identify what worked, what failed, and what surprised the team.
3. Convert lessons into concrete follow-ups.
4. Note any framework improvements.

## Expected Outputs

- Filled `templates/retrospective.md`

## Quality Bar

The retro should change future behavior, not merely describe the past.

## Common Failure Modes

- Writing vague lessons.
- Avoiding uncomfortable causes.
- Creating follow-ups without triggers or owners.

## Relationship To Other HVN Skills And Commands

Feeds future onboarding, planning, and framework updates.
