# HVN Aesthetic Profile

## What This Skill Is

A taste-continuity workflow for product-specific visual and writing direction.

## Trigger

Use before design-heavy work, writing-heavy work, redesigns, product launches, and authenticity preflight.

## Do Not Trigger

Do not use for backend-only work or when an existing approved profile already fully covers the task.

## Required Inputs

- Product or issue context
- Profile goal or active profile

## Optional Inputs

- Screenshots
- Brand notes
- Writing samples
- Existing design system

## Exact Workflow

1. Identify active product profile if one exists.
2. Select a shipped profile or create a project-specific one.
3. Record visual tone, layout, typography, motion, writing tone, anti-patterns, and trust cues.
4. Feed the profile into design and writing calibration.
5. Record profile changes in Linear or the opt-out record.

## Expected Outputs

- `templates/aesthetic-profile.md`
- Profile selection note

## Quality Bar

The profile should make future design and writing decisions more consistent.

## Common Failure Modes

- Mixing profiles without a primary direction.
- Treating profile as decoration instead of constraints.
- Changing a shared profile without approval.

## Relationship To Other HVN Skills And Commands

Feeds `hvn-authenticity-preflight`, `hvn-design`, `hvn-review`, and `hvn-style`.
