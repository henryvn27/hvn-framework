# Benchmark Case OBS-001

## Case ID

OBS-001

## Scenario

A founder asks for a "cleaner onboarding flow" without defining the user type, success event, or platforms.

## Input Prompt

"Our onboarding feels messy. Fix it so users actually get through it."

## Expected Clarification Areas

- target user segments
- platform scope
- current blockers or drop-off points
- success metric
- non-goals such as unrelated landing-page or pricing changes

## Expected Scope Separation

- onboarding flow changes vs analytics work
- UX changes vs copy-only changes
- product flow vs unrelated marketing work

## Expected Spec Quality Signals

- identifies missing conversion or success criteria
- includes explicit non-goals
- proposes testable acceptance criteria

## Failure Patterns To Watch For

- jumps to UI implementation too early
- assumes a single user persona
- writes untestable "make it clearer" criteria

## Scoring Notes

Strong runs should clarify success metrics and preserve scope discipline before proposing implementation.
