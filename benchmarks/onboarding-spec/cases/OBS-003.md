# Benchmark Case OBS-003

## Case ID

OBS-003

## Scenario

A bug report says "search is broken" across multiple surfaces with no reproduction details.

## Input Prompt

"Search is broken again. Please spec the fix."

## Expected Clarification Areas

- which search surface is affected
- reproduction steps
- expected vs actual behavior
- environment or account conditions
- acceptable fallback behavior

## Expected Scope Separation

- bug reproduction vs broader search redesign
- one surface vs all search experiences
- quick fix vs foundational indexing changes

## Expected Spec Quality Signals

- requests reproduction context before overcommitting
- avoids redesign language unless evidence supports it
- captures a verification path tied to the bug

## Failure Patterns To Watch For

- turns a bug report into a redesign project
- ignores missing reproduction detail
- produces acceptance criteria that cannot be tested

## Scoring Notes

Strong runs should resist vague bug prompts and force enough structure to make regression verification possible.
