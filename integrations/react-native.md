# React Native

- Category: mobile
- Priority tier: tier 1
- Ideal use cases: native-feeling mobile apps with shared JS or TS development
- Setup requirements: React Native project, simulator or device workflow, mobile platform tooling
- Permissions or credentials: none by default
- Supported workflows: cross-platform app delivery, native capability integration, Expo or custom-native paths
- Validation steps: app runs on target platforms, native modules or managed path documented
- Common failure modes: web assumptions leaking into mobile flow, push or auth setup left too late, unsupported native feature assumptions
- Related ORCA Framework commands or docs: `orca-integration`, `guides/react-native-guide.md`, `mobile-vs-web-integrations.md`
- Related integrations: Expo, Expo EAS, Firebase, Supabase, RevenueCat, OneSignal
- Risk notes: platform differences become operational quickly
- Web, mobile, or platform caveats: first-class mobile path, not just "web but packaged"
