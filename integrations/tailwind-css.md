# Tailwind CSS

- Category: web and product
- Priority tier: tier 2
- Ideal use cases: fast UI iteration, design-system utility workflows
- Setup requirements: package install, config, content paths
- Permissions or credentials: none
- Supported workflows: styling, component systems, design refinement
- Validation steps: classes compile, content paths cover app, production CSS not stripped incorrectly
- Common failure modes: missing content globs, inconsistent design tokens, utility sprawl
- Related ORCA Framework commands or docs: `orca-design`, `web-stack-guide.md`
- Related integrations: Next.js, Vercel
- Risk notes: easy to ship inconsistent design if tokens are weak
- Web, mobile, or platform caveats: web-focused; do not confuse with native mobile styling
