# Mixpanel

- Category: analytics and monitoring
- Priority tier: tier 3
- Ideal use cases: product analytics and funnel analysis
- Setup requirements: project, SDK install, event plan
- Permissions or credentials: project token or credentials
- Supported workflows: event analytics and reporting
- Validation steps: SDK present, event plan exists, test events appear
- Common failure modes: token configured but no useful event taxonomy, duplicate analytics stacks with no reason
- Related ORCA-HVN commands or docs: `orca-integration`, `integration-validation.md`
- Related integrations: Next.js, Expo
- Risk notes: analytics value depends on event quality, not vendor presence
- Web, mobile, or platform caveats: cross-platform with platform-specific event differences
