# Crashlytics

- Category: analytics and monitoring
- Priority tier: tier 3
- Ideal use cases: mobile crash reporting in Firebase-centered apps
- Setup requirements: Firebase setup, SDK integration
- Permissions or credentials: Firebase project access
- Supported workflows: crash capture and mobile runtime diagnostics
- Validation steps: SDK configured, crash report arrives, app linked to Firebase
- Common failure modes: SDK partially configured, duplicate crash stack with no clear ownership
- Related ORCA Framework commands or docs: `orca-integration`, `mobile-stack-guide.md`
- Related integrations: Firebase, React Native, Expo
- Risk notes: choose a primary crash workflow intentionally
- Web, mobile, or platform caveats: mobile-centric monitoring
