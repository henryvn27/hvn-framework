# Playwright

- Category: testing and CI
- Priority tier: tier 2
- Ideal use cases: browser testing, web QA, regression coverage
- Setup requirements: package install, browsers, test config
- Permissions or credentials: none by default
- Supported workflows: end-to-end tests, smoke tests, QA automation
- Validation steps: tests run locally, target app reachable, config matches environment
- Common failure modes: flaky selectors, env mismatch, tests without stable fixtures
- Related ORCA-HVN commands or docs: `orca-test-blind`, `orca-test-regression`, `orca-integration`
- Related integrations: GitHub Actions, Next.js
- Risk notes: flaky test suites burn trust quickly
- Web, mobile, or platform caveats: web-first browser automation
