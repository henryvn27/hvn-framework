# Next.js

- Category: web and product
- Priority tier: tier 1
- Ideal use cases: web SaaS, app dashboards, marketing plus product surfaces
- Setup requirements: Node toolchain, app structure, deployment target
- Permissions or credentials: none by default
- Supported workflows: app routing, SSR or edge patterns, web product delivery

- Common failure modes: unclear deploy assumptions, auth middleware drift, SSR versus client confusion
- Related ORCA-HVN commands or docs: `orca-integration`, `web-stack-guide.md`, `saas-stack-guide.md`
- Related integrations: Vercel, Tailwind CSS, Supabase, Clerk, Stripe
- Risk notes: hosting assumptions matter
- Web, mobile, or platform caveats: web-first; different concerns than Expo or React Native
