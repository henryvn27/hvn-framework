# Command Index

Use this page when you know what you want to do but not which command matches it.

If you are new, start with [first-workflow.md](first-workflow.md) before using this page as a catalog.

The `orca-*` entries below are ORCA workflow command definitions.
After install, matching `orca-*` shims are available from the installed `bin/` directory when that path is added to your shell.

## Start Or Clarify Work

- `orca-install`: guided install path selection and setup
- `orca-doctor`: install and setup validation
- `orca-check-updates`: inspect update availability and risk before applying changes
- `orca-corpus`: configure or inspect persistent corpus support
- `orca-onboard`: clarify a work item
- `orca-discover`: inspect the repo and constraints
- `orca-legacy`: start with archaeology when the codebase is inherited or fragile
- `orca-idea`: start an opportunity before product planning
- `orca-demo`: ask a few questions, pick a personalized demo direction, and generate a real ORCA `/goal` prompt

## Plan The Work

- `orca-spec`: define the product or implementation contract
- `orca-plan`: build the implementation phases
- `orca-next`: get short default next-step guidance
- `orca-goal`: decide whether the next milestone should use goal mode
- `orca-integration`: pick the right integration or stack pack
- `orca-impeccable`: route frontend and design work into the official Impeccable wrapper path
- `orca-superpowers`: route execution-heavy coding work into the official Superpowers wrapper path
- `orca-recommend-stack`: recommend a stack only when the fit is strong
- `orca-setup-integration`: set up a chosen integration path
- `orca-learning`: tune adaptive learning-help behavior
- `orca-feedback`: ask for or suppress a specific coaching response
- `orca-explain`: open a dedicated explanation session for the current work
- `orca-btw`: use the harness's quickest real side-question path when available, otherwise ask a framed non-mutating side question

## Run The Work

- `orca-build`: implement approved scope
- `orca-background`: define a bounded unattended run
- `orca-keep-going`: continue current work in background mode
- `orca-delegate`: hand bounded work to another agent or harness
- `orca-ui-debug`: run persistent interactive UI debugging
- `orca-web-qa`: drive a real browser for QA and artifact capture

## Inspect, Coordinate, And Recover

- `orca-status`: inspect runtime and workflow state
- `orca-explain`: understand what is happening and ask follow-up questions without leaving the workflow
- `orca-btw`: ask one quick side question without turning it into a new execution thread
- `orca-context`: resolve an ambiguous artifact, app, or recent-work reference before deeper work starts
- `orca-update`: verify, apply, defer, or recover from ORCA updates
- `orca-background-status`: inspect current or last unattended run
- `orca-inspect`: review run identity, artifacts, and blockers
- `orca-receipt`: write a compact run summary
- `orca-lineage`: connect related artifacts
- `orca-replay`: compare behavior across versions or runtime changes
- `orca-restore`: recover from a known-good workflow state

## Safety And Governance

- `orca-approve`: explicit approval flow
- `orca-checkpoint`: pause and request a human decision
- `orca-tool-review`: evaluate a risky tool
- `orca-mcp-review`: evaluate a new MCP server
- `orca-security-check`: inspect risky security surfaces

## QA And Improvement

- `orca-review`: review implementation quality
- `orca-design`: review design quality, hierarchy, copy, and generic-AI-looking risk
- `orca-ci`: inspect failing CI checks and extract actionable failure context
- `orca-pr-feedback`: organize PR comments and review threads into a remediation queue
- `orca-test-blind`: first-look QA
- `orca-test-briefed`: bounded-context retest
- `orca-test-regression`: regression confirmation
- `orca-screenshot`: capture app, browser, or desktop screenshot evidence
- `orca-regression-task`: preserve repeated findings
- `orca-eval`: evaluate workflow behavior
- `orca-benchmark`: compare framework quality over time
- `orca-improve-framework`: turn reusable session friction or strong quality-signal evidence into a framework backlog or issue draft

## Docs And Navigation

- `orca-docs`: route a docs question or plan a docs refresh
- `orca-schema`: inspect portable artifact structure
- `orca-help`: general command overview
