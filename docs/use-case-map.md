# Use-Case Map

The paths below use ORCA workflow command names.
Unless your host adds explicit ORCA slash commands, use these as prompt structure rather than shell commands.

## I Need To Install ORCA

Recommended path:

`orca-install` -> `orca-doctor` -> `docs/first-run.md`

## I Need To Update ORCA Safely

Recommended path:

`orca-check-updates` -> `orca-update` -> `orca-doctor`

## I Need To Ship A New Feature

Recommended path:

`orca-onboard` -> `orca-spec` -> `orca-plan` -> `orca-build` -> `orca-review` -> `orca-test-blind`

## I Inherited A Messy Repo

Recommended path:

`orca-legacy` -> `orca-discover` -> `orca-spec` -> `orca-plan`

## I Want A Controller Agent To Manage A Coding Agent

Recommended path:

`orca-controller` -> `orca-orient` -> `orca-delegate` -> `orca-ingest`

## I Want Bounded Work While I Am Away

Recommended path:

`orca-background` or `orca-keep-going` -> `orca-background-status`

## I Need A Long-Running Milestone Contract

Recommended path:

`orca-goal` -> `orca-goal-create` -> `orca-goal-status` -> `orca-goal-review`

## I Need To Evaluate A Startup Or Product Idea

Recommended path:

`orca-idea` -> `orca-evaluate-idea` -> `orca-plan-idea` -> `orca-validate-idea`

## I Need To Understand Harness Differences

Recommended path:

`orca-runtime` -> `orca-status` -> `docs/compatibility-matrix.md`
