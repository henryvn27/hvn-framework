# Improvement Signals

Improvement should be driven by explicit signal classes.

## Signal Classes

- user preference signal
- repeated workflow signal
- repeated blocker signal
- docs confusion signal
- recommendation mismatch signal
- quality or frustration signal
- successful intervention signal
- integration demand signal
- repeated manual-work signal
- orchestration failure signal
- ecosystem shift signal

## For Each Signal

Track:

- local or global scope
- confidence
- evidence needed
- likely action path
