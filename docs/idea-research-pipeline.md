# Idea Research Pipeline

This optional pipeline gives ORCA-HVN a repeatable way to evaluate an idea without collapsing into generic brainstorming.

## Sequence

1. Capture the idea in [templates/idea-intake.md](../templates/idea-intake.md).
2. Compress it into [templates/idea-one-pager.md](../templates/idea-one-pager.md).
3. Generate evaluation questions from the scorecard and evidence map.
4. Run targeted desk research on market, alternatives, and customer behavior.
5. Write an opportunity memo.
6. Decide whether to pursue, park, kill, or revise.
7. If still alive, design the next validation experiment.
8. If validation support is strong enough, bridge into `orca-spec`.

## Folder Structure

When the user wants local artifacts, store:

- one-pager
- scorecard
- evidence map
- research briefs
- opportunity memo
- validation plan
- decision note

in one durable idea folder so later product work can inherit the context.

## Success Condition

The pipeline is successful when it reduces uncertainty enough to change the next action, not when it produces the most polished memo.
