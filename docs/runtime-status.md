# Runtime Status

Runtime status is the inspectable summary of why HVN is behaving a certain way in the current harness.

## It Should Show

- detected harness
- detection confidence
- enabled features
- disabled features
- degraded capabilities
- active policy switches
- recommended workflow mode
- missing integrations
- important caveats

## Purpose

Use runtime status to make routing decisions visible to users and maintainers. This reduces mystery and makes incorrect adaptation easier to spot.
