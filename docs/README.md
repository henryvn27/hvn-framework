# Documentation Home

![HVN orca mascot](assets/orca.png)

HVN documentation is organized as a layered system.
Start with the overview pages, then move into guides and reference docs based on your role and current task.

## Start Here

- New to HVN: [start-here.md](start-here.md)
- Want the short version: [quickstart.md](quickstart.md)
- Need the mental model first: [intro.md](intro.md)
- Prefer a wiki-style table of contents: [../wiki/Home.md](../wiki/Home.md)

## Layer Map

- Intro: what HVN is, who it is for, and how the pieces fit together
- Quickstart: the shortest path to first value
- Concepts: mental models, glossary, architecture, and workflow
- Guides: task-oriented reading for a specific host or mode
- Indexes: routing pages for features, commands, and use cases
- Reference: detailed behavior docs, command docs, skills, install docs, and host specifics
- Contributors: documentation maintenance rules, ownership, freshness signals, and automation
- What's new: recent changes and rediscoverability pages

See [information-architecture.md](information-architecture.md) for the canonical map.
