# Orchestration Observability

Multi-agent behavior should be inspectable, not magical.

## Track

- why delegation happened
- which pattern was chosen
- what workers were invoked
- context packet size and scope
- retries
- blockers
- approvals
- result quality
- whether orchestration helped or hurt

## Core Questions

The observability layer should make it possible to answer:

- was the worker worth it
- did delegation improve quality or speed
- where did orchestration fail
