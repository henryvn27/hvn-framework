# Instance Improvement Loop

Loop A improves the user's own ORCA instance.

## What It Learns From

- repeated user preferences
- repeated workflow patterns
- chosen tools and integrations
- recurring clarification needs
- successful prompt and context patterns
- skill-level adaptation
- docs and help preferences
- repeated local friction
- local automation opportunities
- vault and memory patterns

## What It May Improve

- local defaults
- guidance style
- recommended workflow path
- local docs and help routing
- tool or setup support behavior
- onboarding help level
- background automation suggestions
- memory extraction patterns

## Risk Rule

Local improvements should feel stable and useful, not jittery.

Do not churn local behavior based on one weak signal.
