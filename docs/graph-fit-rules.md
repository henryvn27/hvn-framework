# Graph Fit Rules

Graph or vault analysis is useful only when it improves understanding enough to justify the extra step.

## Strong Fit

Graph support is a strong fit when:

- the vault is large or fragmented
- related ideas are scattered across many notes
- duplicate concepts exist under different names
- project or workflow clusters are hard to see
- onboarding to the vault is confusing
- the user explicitly wants a map of their knowledge system

## Possible Fit

Graph support may help when:

- the user suspects duplicated docs or weak structure
- wiki or hub pages feel incomplete
- the work depends on seeing cross-project themes

## Weak Fit

Graph support is probably not necessary when:

- the task is one-off setup help
- the vault is small and already clear
- the current question is narrow
- basic vault inspection is already enough

## Recommendation Rule

When graph support is only a possible fit, suggest it lightly:

`This could help if you want a better map of your vault, but it is optional.`
