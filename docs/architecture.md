# Architecture

HVN is organized around four installable layers:

- `HVN.md` is the master operating manual.
- `commands/` contains entry-point prompts for specific workflows.
- `skills/` contains reusable procedures that commands invoke.
- `templates/` contains durable artifacts generated by commands and skills.

Docs explain the system, examples show finished artifacts, and scripts validate that the repository remains coherent.

## Why Commands And Skills Are Separate

Commands answer "what workflow should start now?" Skills answer "how should this behavior be performed?" A command can call multiple skills, and a skill can support multiple commands.

## Why Blind QA Is Separate

Blind QA is intentionally separated from informed QA because hidden knowledge changes the tester's behavior. Once a tester knows the intended flow, confusing UI may look obvious. HVN protects the first-look pass by forbidding hidden context until the blind report is complete.

## Context Briefer

The context briefer creates bounded packets for retesting. It lists exactly what is included and withheld. This allows layered QA:

- Blind pass for first impression
- Briefed pass for target behavior
- Informed pass for implementation-level verification

The briefer preserves first-look integrity by preparing context only after the blind report is saved.
