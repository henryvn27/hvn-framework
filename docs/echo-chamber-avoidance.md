# Echo-Chamber Avoidance

Background mode should not become a system that keeps rephrasing its own prior thinking.

## Core Rule

Each iteration must introduce one of:

- new evidence
- new structure
- new artifact state
- new verification output
- a narrower constraint

If not, stop.

## Examples Of Echo-Chamber Behavior

- repeating the same plan in slightly different words
- restating the same blocker without changing scope or approach
- running the same low-value check without new inputs
- revisiting the same speculation after no new evidence has appeared

## How To Avoid It

- link each step to a concrete expected delta
- prefer artifact changes over explanatory restatement
- require receipts to name what changed since the previous step
- use runtime status, lineage, and prior receipts as a baseline before continuing

## Exit Rule

If the run cannot explain what changed since the last step, it should choose a final state and stop.
