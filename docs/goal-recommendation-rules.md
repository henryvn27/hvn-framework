# Goal Recommendation Rules

HVN may recommend goal mode when all required conditions are met.

## Required Conditions

- approved spec exists
- milestone or phase is bounded
- success condition is measurable
- one clear verification contract exists
- verification path exists
- task is long enough to benefit from persistence
- safety level is acceptable

## Strong Signals

- repeated manual nudges would add little value
- the work fits inside one milestone-sized outcome
- verification can be run by the agent
- blockers can be recognized clearly
- the work can stop at a reviewable boundary

## Do Not Recommend Goal Mode

Do not recommend goal mode when:

- the task is still discovery-heavy
- the issue needs product clarification
- approval is unresolved
- verification cannot be observed
- the goal would span unrelated systems
- the goal is project-sized with fuzzy outcomes
