# Framework Improvement Loop

Loop B improves ORCA as a project.

## What It Learns From

- repeated session-level friction
- repeated issue candidates
- docs drift or confusion
- weak paved roads
- bad defaults
- orchestration failures
- poor setup guidance
- repeated quality problems
- repeated generic-output or wrong-direction signals
- ecosystem changes
- recurring integration gaps

## What It May Improve

- framework docs
- onboarding
- recommendation policy
- paved roads
- orchestration guidance
- integration packs
- prompts
- issue templates
- automation and validation
- visibility and disclosure rules

## Evidence Rule

Global changes require stronger evidence than local adaptations.

One user's preference is not enough to rewrite the framework default.
