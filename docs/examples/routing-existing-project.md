# Routing Example: Existing Project

## Request

Improve the settings page in an existing app.

## HVN Routing

1. Non-trivial work: use Linear.
2. Primary task: improve existing product UI.
3. Routing decision: route to an improve-in-place or redesign-existing-projects skill if installed.
4. Preserve: framework, navigation, state model, design-system primitives, and test conventions.
5. Redesign only if the user explicitly requests it or the approved spec requires it.

## Outcome

The agent audits the existing page, identifies targeted upgrades, posts a plan to Linear, waits for approval when needed, and changes only the relevant UI surface.
