# Web App Example

This example sequence fits a production web app feature.

1. `orca-onboard` gathers user, task, success, and constraint context.
2. `orca-discover` inspects routes, components, data flow, and test setup.
3. `orca-spec` writes acceptance criteria for the user-facing flow.
4. `orca-plan` breaks the change into implementation and verification phases.
5. `orca-build` implements the feature.
6. `orca-review` checks behavior and maintainability.
7. `orca-test-blind` opens the app in a browser with no hidden context.
8. `orca-test-briefed` retests fixes using a context packet.
9. `orca-ship` prepares release notes and final checks.
