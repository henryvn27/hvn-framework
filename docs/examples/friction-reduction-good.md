# Friction Reduction Good

The user wants to ship a small internal tool and already uses GitHub and Linear.

ORCA Framework should:

- recommend one clear internal-tool road
- use the existing GitHub and Linear setup if available
- avoid suggesting extra knowledge or automation tooling
- keep the first path to `orca-onboard -> orca-spec -> orca-plan -> orca-build -> orca-review`

Why it is good:

- fewer choices
- no extra setup project
- strong-fit recommendation only
