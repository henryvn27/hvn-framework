# iOS App Example

This example sequence fits an iOS feature using simulator QA.

1. `hvn-onboard` clarifies the user journey and platform constraints.
2. `hvn-discover` inspects SwiftUI views, navigation, state, and build settings.
3. `hvn-spec` defines expected behavior and accessibility expectations.
4. `hvn-plan` identifies simulator verification steps.
5. `hvn-build` implements the feature.
6. `hvn-test-blind` launches the app through the simulator MCP with no hidden context.
7. `hvn-test-briefed` retests known flows with a bounded packet.
8. `hvn-test-regression` verifies neighboring flows.
