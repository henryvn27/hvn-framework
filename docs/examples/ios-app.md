# iOS App Example

This example sequence fits an iOS feature using simulator QA.

1. `orca-onboard` clarifies the user journey and platform constraints.
2. `orca-discover` inspects SwiftUI views, navigation, state, and build settings.
3. `orca-spec` defines expected behavior and accessibility expectations.
4. `orca-plan` identifies simulator verification steps.
5. `orca-build` implements the feature.
6. `orca-test-blind` launches the app through the simulator MCP with no hidden context.
7. `orca-test-briefed` retests known flows with a bounded packet.
8. `orca-test-regression` verifies neighboring flows.
