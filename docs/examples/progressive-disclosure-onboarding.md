# Progressive Disclosure Onboarding

First-run onboarding should:

- explain the current phase in one or two lines
- point the user to one next command
- defer advanced concepts until a real need exists

Good sequence:

1. start-here
2. intro guide
3. first workflow
4. contextual guides only when needed

Bad sequence:

1. show the full command catalog
2. explain every optional integration
3. make the user choose among advanced modes before the first spec
