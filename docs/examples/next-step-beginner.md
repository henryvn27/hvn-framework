# Next-Step Example: More Context

Situation: the spec was completed and the user wants a clear next move.

```text
What just happened: the spec now defines the goal, boundaries, and acceptance criteria.

Next: split it into one or more milestones before building. That keeps the first implementation slice small enough to verify.

Optional: if any acceptance criteria still feel unclear, revise the spec first.

Done for the next phase means each milestone has a clear scope and a way to prove it works.
```

This gives context without assuming the user lacks skill.
