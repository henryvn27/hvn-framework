# Onboarding Session Example

## Prompt

Build a lightweight app that helps consultants prepare client meeting briefs.

## Adaptive Questions

1. Who prepares the brief and how much time do they have?
2. What inputs are available before a meeting?
3. What does a useful brief contain?
4. Where does the brief need to be exported or shared?
5. What information should never be included?

## Output

- Primary user: consultant preparing a meeting in under ten minutes.
- Core job: convert scattered client context into a concise meeting brief.
- Key risks: confidential data, stale CRM notes, vague recommendations.
- Recommended workflow: onboarding, discovery, spec, build, web QA, security review.
