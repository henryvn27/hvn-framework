# Example: Anti-Generic Writing

## Task

Rewrite a README for a developer tool.

## Writing Calibration

Audience: engineers deciding whether the tool is worth installing.

Voice: direct, concrete, maintainer-like. Short paragraphs. No startup pitch language.

Avoid:

- "Seamlessly empower your workflow"
- Generic productivity claims
- Long explanations of obvious developer pain
- Polished paragraphs that avoid tradeoffs

Use:

- Exact install path
- One real command
- Clear statement of what the tool does not do
- Maintenance and verification notes

## Resulting Direction

The README should read like a maintainer explaining a useful tool, not a landing page trying to convert everyone.
