# NotebookLM For Project Knowledge

NotebookLM can serve as a user-managed project knowledge notebook when the team wants a document-centered reference layer.

Good ORCA-HVN behavior:

- treat NotebookLM as supplemental project knowledge, not a replacement for specs, run memory, or receipts
- use it for long-form reference digestion and question answering
- keep the authoritative execution artifacts in ORCA-HVN
