# Example Hermes Zip Project Entry

1. Read `HVN-STATUS.md`.
2. Read the latest orientation or inspection artifact.
3. Check active phase, blockers, and missing artifacts.
4. Decide whether Hermes should act directly or delegate.
5. If delegating implementation, create a delegation brief for Codex.

## Why This Works

The controller does not need to infer project state from scattered prose. ORCA-HVN exposes a concise entry surface first, then deeper artifacts.
