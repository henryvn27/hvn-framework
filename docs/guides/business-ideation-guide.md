# Business Ideation Guide

Use this guide when the starting point is a rough product idea, not an implementation-ready task.

## When To Use It

- The product problem is clearer than the solution
- You need structured intake before architecture or build choices
- You want to separate v1 from later ideas early

## Recommended Flow

1. Start with `hvn-onboard` or `hvn-question-flow`.
2. Gather target user, problem, constraints, and success criteria.
3. Convert the intake into a spec only when the shape of the first release is visible.
4. Split v1, later-phase, and out-of-scope ideas before planning.

## Common Pitfalls

- Asking for implementation details before the user job is clear
- Collapsing every good idea into the first release
- Mistaking brainstorming for acceptance criteria
