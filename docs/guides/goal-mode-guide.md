# Goal Mode Guide

Use this guide when the work is oriented around a concrete outcome with a clear done state.

## When To Use Goal Mode

- A feature, bug fix, or rollout has a crisp target
- The work item can be evaluated against acceptance criteria
- You need a spec, plan, and verification path

## Recommended Flow

1. Clarify the outcome with onboarding or question flow if needed.
2. Write the spec.
3. Split phases only after the spec is credible.
4. Execute against the spec and verify the result.

## Common Pitfalls

- Treating a vague aspiration as a goal
- Turning goal mode into task spam before the problem is clear
- Declaring done without review or QA evidence
