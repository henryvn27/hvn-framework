# Paved Roads Guide

Use this guide when you want repeatable, low-ambiguity execution for common work patterns.

## What A Paved Road Means In HVN

A paved road is a recommended path with explicit artifacts, routing expectations, and verification gates.
It reduces re-decision for common scenarios without pretending every project is identical.

## Good Candidates

- New feature intake to spec
- Spec to plan
- Build to review to QA
- Install and onboarding for a new host

## Common Pitfalls

- Treating the paved road as inflexible even when the task changes
- Adding a new bespoke doc instead of improving the existing path
- Forgetting to update indexes and wiki routes when the paved road changes
