# Using ORCA-HVN With gstack

## When To Use This Path

Use this guide when gstack is the main working surface around your execution flow.

## How It Works

gstack can act as the fast working layer for browsing, QA, and adjacent workflow support while ORCA-HVN supplies routing, structure, receipts, and compatibility-aware guidance.

## Recommended Flow

1. `orca-runtime`
2. `orca-onboard`
3. `orca-spec`
4. `orca-plan`
5. `orca-build`
6. `orca-review` and QA commands

## Common Pitfalls

- assuming a gstack-centered workflow replaces the need for runtime and workflow discipline
- skipping receipts and lineage because the code changed locally
- using background mode without a bounded contract
