# Idea One-Pagers

The one-pager is the default artifact for early ORCA Framework ideation. It forces an idea into a shape that can be challenged without a long pitch deck.

## Why ORCA Framework Starts Here

A one-pager is useful because it makes weak thinking visible quickly:

- unclear target user
- vague problem statements
- missing current alternatives
- no reason why now
- no founder angle
- no next validation step

If an idea cannot survive one page, it is usually too early for roadmap or build planning.

## Required Shape

Use [templates/idea-one-pager.md](../templates/idea-one-pager.md). The artifact should cover:

- idea name
- one-sentence description
- target customer
- painful problem
- current alternatives
- proposed solution
- why now
- founder or team angle
- major assumptions
- open questions
- next validation step

## Writing Rules

- Prefer specific users over broad categories.
- Prefer current behavior over abstract demand claims.
- Prefer crisp assumptions over hidden optimism.
- Name the next decision the team is trying to make.
- Stop before it turns into a strategy essay.

## Good Outcome

A good one-pager should make it easy for another person to say:

- what the idea is
- who it is for
- why the problem matters
- what is still unproven
- what should be tested next
