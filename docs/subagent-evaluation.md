# Subagent Evaluation

Subagent usage should be evaluated the same way other framework behaviors are evaluated: by outcome, not by novelty.

## Evaluate

- task success
- context efficiency
- retry burden
- tool-call efficiency
- quality uplift
- parent coordination cost
- escalation clarity
- merge friction
- ownership confusion

For install or onboarding races, also evaluate:

- time to completion
- required human replies
- avoidable hassle
- information captured versus questions asked
- whether the subagent respected the mission instead of expanding scope

## Rule

If a subagent repeatedly adds complexity without improving outcomes, ORCA Framework should demote that pattern.
