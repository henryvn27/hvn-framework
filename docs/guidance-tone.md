# Guidance Tone

Next-step guidance should be calm, concise, adaptive, actionable, skippable, and non-patronizing.

## Rules

- Never condescend.
- Never assume ignorance.
- Never show off expertise.
- Never over-explain obvious things to advanced users.
- Never hide necessary context from beginners.
- Avoid filler and fake cheerleading.
- Use plain language.
- Use structure, not fluff.
- Prefer direct verbs over motivational language.
- Keep jargon out unless the user or repo already uses it.

## Preferred Shape

Use:

- one clear default next step
- one optional alternate path
- one sentence explaining why

Avoid:

- long tutorials after every phase
- "as a beginner" phrasing
- "obviously" or "simply" phrasing
- generic praise
- process theater

## Respectful Adaptation

Beginner guidance can include context without sounding childish. Advanced guidance can be terse without becoming cryptic. The user should feel oriented, not managed.
