# Installation Modes

## Repo Mode

Repo mode uses this repository directly. It is best for contributors and teams that vendor HVN into a product repo.

## Local Mode

Local mode installs HVN into a project folder, usually `./.hvn`. It is best when a single product wants a stable copy of the framework.

## Global Mode

Global mode installs HVN into `$HOME/.hvn`. It is best when one user wants the same commands and skills available across projects.

## Choosing A Mode

Use repo mode while editing HVN itself. Use local mode for team projects that want committed framework artifacts. Use global mode for personal agent configuration.
