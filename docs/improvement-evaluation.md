# Improvement Evaluation

Every meaningful improvement proposal should say:

- what problem it addresses
- what evidence supports it
- what change is proposed
- what better means
- how success will be judged
- when to stop or roll back

## Useful Success Measures

- fewer clarification loops
- fewer repeated setup questions
- fewer duplicate docs
- faster project orientation
- better issue quality
- better handoff success
- fewer wrong-tool suggestions
- better satisfaction signals
- fewer repeated blockers
