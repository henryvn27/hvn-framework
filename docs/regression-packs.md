# Regression Packs

Regression packs turn bugs, confusion points, review findings, and QA failures into reusable retest scenarios.

## Purpose

Regression packs prevent teams from losing hard-won QA knowledge after a fix ships.

## Sources

Create packs from:

- Blind QA findings
- Guided QA failures
- Review findings
- Bug reports
- Incidents
- Reopened issues

## Contents

A regression pack captures:

- Source issue or finding
- Affected platform
- Reproduction steps
- Expected behavior
- Actual behavior
- Risk level
- Fix verification steps
- Future retest scenario
- Related artifacts or screenshots

## Reuse

Before ship, run relevant regression packs for the touched product area. Link results back to the issue.

## Linear Integration

Attach or link regression packs to the Linear issue and project. Use labels or linked artifacts so future agents can discover them.
