# Concept Map

HVN is easiest to understand as a set of connected layers.

## The Shape

1. A work item exists in Linear or an opt-out equivalent.
2. Onboarding and question flow clarify the problem.
3. Discovery and research reduce unknowns.
4. A spec defines the contract.
5. A plan splits the contract into phases.
6. Build, review, QA, and ship verify the result.
7. Run memory keeps the work resumable.
8. Host adapters let the same model run in different environments.

## Key Relationships

- Harnesses host the workflow but do not redefine it.
- Commands start workflows; skills define how workflows behave.
- Run memory links artifacts but does not replace them.
- Blind QA protects first impressions; briefed QA validates intended behavior.
- Documentation routes users into the right layer instead of forcing deep docs too early.

## Concept Families

- Work management:
  - work item, state, project, issue health, receipt
- Artifact discipline:
  - onboarding intake, spec, plan, report, checklist, run memory
- Execution posture:
  - goal mode, background mode, paved road, shadow mode
- Host model:
  - harness, runtime adaptation, compatibility matrix
