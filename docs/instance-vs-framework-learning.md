# Instance Vs Framework Learning

ORCA should keep local adaptation and global evolution separate.

## Local Instance Learning

Use local learning for:

- one user's preferences
- one user's docs habits
- one user's preferred explanation depth
- repeated local workflow patterns
- local tool and integration choices

## Framework Learning

Use framework learning for:

- repeated cross-session friction
- repeated docs confusion
- bad defaults
- weak paved roads
- repeated orchestration failures
- recurring setup pain

## Separation Rule

If the evidence is mostly about one user's style or one local setup, keep it local.

If the evidence is recurring, generalizable, and framework-shaped, it may become a framework candidate.
