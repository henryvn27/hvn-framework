# Glossary

This glossary defines common HVN terms in plain language.

## Harness

A host runtime or agent environment such as Claude Code, Codex CLI, OpenCode, or Hermes Agent.

## Controller Agent

An agent that coordinates work, routing, or delegation rather than executing every step directly.

## Executor Agent

An agent that performs a bounded implementation, review, QA, or research task.

## Goal Mode

A work posture where the outcome and done state are concrete enough to drive a spec and verification plan.

## Paved Road

A recommended repeatable path for a common workflow with explicit artifacts and gates.

## Compatibility Matrix

A document that explains what behavior is shared across hosts and what differs by runtime.

## Stage Budget

The practical amount of scope, effort, or context a single workflow stage should absorb before it should split or stop.

## Shadow Mode

A way of evaluating or dry-running a workflow without treating the output as the live source of truth yet.

## Artifact Lineage

The chain that connects onboarding, spec, plan, implementation, review, QA, and ship artifacts for one work item.

## Background Mode

A continuity-oriented mode for work that spans sessions, handoffs, or multiple agents.

## Idea Validation

Early work that tests whether a product or feature idea is real enough to justify spec and build effort.

## Receipt

A compact durable record of what was done, what changed, and what comes next.

## Runtime Adaptation

The host-specific adjustments HVN makes while preserving one shared workflow model.

## Run Memory

A compact continuation artifact that helps a new agent resume the current workstream without replaying a full transcript.

## Blind QA

A first-look test pass done without hidden implementation context.
