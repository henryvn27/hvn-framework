# Linear Defaults

HVN defaults to Linear for non-trivial work.

## What Counts As Non-Trivial

Use Linear for:

- Projects
- Features
- Bug fixes
- Research tasks
- Multi-step changes
- Review or QA efforts
- Release preparation
- Work requiring handoff

Tiny one-line answers or local commands do not require Linear.

## Project Rule

At the start of non-trivial work:

1. Inspect Linear for a project matching the current repository, app, product, or client effort.
2. Reuse the matching project if present.
3. Create a separate project if none exists.
4. Keep unrelated efforts in separate projects.

## Issue Rule

Create or reuse a Linear issue for each meaningful unit of work. The issue should hold scope, plan, blockers, status, links, review findings, QA reports, and ship readiness.

## Status Rule

Keep issue status and comments current. Every meaningful agent pass should leave a concise comment or prepared comment when Linear is blocked.

## Blocked Connector Rule

When Linear access is unavailable:

- Say Linear is blocked.
- Continue local work only when appropriate.
- Record what project should exist.
- Record what issue should exist or be updated.
- Record the exact comment, labels, links, and status recommendation for later sync.

## Summary Rule

Final handoffs should include Linear project and issue identifiers when available. If unavailable, include the pending Linear sync.
