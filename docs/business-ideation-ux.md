# Business Ideation UX

The ORCA Framework ideation lane should work for both exploratory users and serious founders without splitting into patronizing beginner and expert versions.

## Interaction Modes

Use progressive disclosure:

- guided mode when the user needs structure
- fast mode when the user already thinks in startup or strategy primitives

Both modes should produce the same core artifacts.

## Guided Mode

Guided mode should:

- ask for one concrete input at a time
- force the user to name customer, pain, alternatives, and next question
- expose missing evidence early
- keep output compact

Use [templates/idea-guided-mode.md](../templates/idea-guided-mode.md).

## Fast Mode

Fast mode should:

- let a strong operator draft the one-pager quickly
- move directly into scoring, memo, or validation planning
- preserve the same honesty rules

Use [templates/idea-fast-mode.md](../templates/idea-fast-mode.md).

## UX Rules

- Prefer one strong default workflow.
- Avoid giant menus of equal options.
- Keep artifacts short enough to revise quickly.
- Make kill and park decisions feel legitimate, not like failure states.
