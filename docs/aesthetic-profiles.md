# Aesthetic Profiles

Aesthetic profiles preserve visual and writing direction across issues. They prevent each agent from re-deciding product taste from scratch.

## Purpose

Profiles define:

- Visual tone
- Layout tendencies
- Typography tendencies
- Motion tendencies
- Writing tone
- Anti-patterns to avoid
- Trust and brand cues
- How to influence design and writing calibration

## Choosing A Profile

Pick one primary profile for a product or issue. Mix profiles only when the tradeoff is explicit. For example, a systems dashboard can borrow editorial writing constraints, but its layout should still prioritize operational scanning.

## Project-Specific Profiles

Create a project-specific profile when a product has stable taste decisions. Store it in `profiles/` or link it from the Linear project.

## Interaction With Authenticity Preflight

Design and writing calibration should read the active aesthetic profile before producing direction. Review should check whether the output stayed consistent with the profile.

## Consistency Across Issues

In Linear-first mode, the active profile should be linked in the project or issue. New issues should reuse it unless a redesign is explicitly approved.
