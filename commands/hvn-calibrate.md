# hvn-calibrate

## Purpose

Run authenticity preflight for work that includes UI, design, frontend presentation, product copy, or polished writing.

## When To Use

Use before high-visibility UI, design-heavy work, landing pages, dashboards, mobile UI, redesigns, UX copy, README work, product pages, docs, essays, outreach, or brand-sensitive writing.

## Required Inputs

- Task description
- Medium: design, writing, or both
- Audience or target user

## Optional Inputs

- Linear issue ID
- Brand constraints
- Existing product references
- Installed design or writing skills

## Linear Context

- Reads: issue scope, audience, product context, labels, and linked references
- Posts: calibration note before execution or as part of the plan
- Trigger: UI, design, product copy, frontend presentation, or polished writing
- Human approval: not required unless calibration changes scope

## Workflow

1. Decide whether design calibration, writing calibration, or both are needed.
2. Run a short targeted research-and-calibration pass.
3. Identify generic patterns to avoid.
4. Define the specific direction for this project.
5. Record review criteria.
6. Post or prepare the calibration for Linear or the opt-out record.

## Outputs And Artifacts

- `templates/design-calibration.md`
- `templates/writing-calibration.md`

## Failure Cases

- If audience or medium is unclear, ask before drafting or building.
- If calibration would be unnecessary bureaucracy for a trivial task, state that it is skipped.

## Related Commands And Skills

- Commands: `hvn-calibrate-design`, `hvn-calibrate-writing`, `hvn-route`
- Skills: `hvn-authenticity-preflight`, `hvn-router`
