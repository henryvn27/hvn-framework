# hvn-calibrate-writing

## Purpose

Run authenticity preflight for polished writing.

## When To Use

Use before README writing, product copy, landing-page copy, docs, essays, thought pieces, outreach, brand writing, or any writing where credibility and voice matter.

## Required Inputs

- Writing task
- Audience
- Medium

## Optional Inputs

- Linear issue ID
- Brand voice
- Source facts
- Examples to match or avoid

## Linear Context

- Reads: issue context, target audience, product facts, and linked references
- Posts: writing calibration note and review criteria
- Trigger: writing-heavy work or `writing-calibration`
- Human approval: not required unless calibration changes scope

## Workflow

1. Identify audience, medium, and desired credibility signal.
2. List generic writing patterns to avoid.
3. Define target voice, rhythm, vocabulary, and point of view.
4. Identify concrete details and examples to include.
5. Record review criteria.

## Outputs And Artifacts

- `templates/writing-calibration.md`

## Failure Cases

- If requested details would require fake anecdotes, refuse that approach and use grounded facts instead.

## Related Commands And Skills

- Commands: `hvn-calibrate`, `hvn-review`
- Skills: `hvn-authenticity-preflight`, `hvn-full-output`
