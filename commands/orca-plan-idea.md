# orca-plan-idea

## Purpose

Turn a promising idea into an opportunity memo, next-step plan, and bridge to later product work.

## When To Use

Use when the idea has survived first-pass evaluation and the user needs a practical plan instead of more vague discussion.

## Required Inputs

- One-pager
- Evaluation notes or scorecard

## Optional Inputs

- Research briefs
- Validation results
- Founder constraints

## Workflow

1. Use `orca-business-ideation`.
2. Review the one-pager, scorecard, and evidence gaps.
3. Write `templates/opportunity-memo.md`.
4. Write `templates/idea-decision.md`.
5. If the idea remains alive, outline the validation path and eventual bridge into `orca-spec`.

## Outputs And Artifacts

- `templates/opportunity-memo.md`
- `templates/idea-decision.md`
- `templates/validation-plan.md` when appropriate

## Failure Cases

- If the idea is still too vague, return to `orca-idea`.
- If the opportunity is weak, do not invent an execution roadmap to rescue it.

## Related Commands And Skills

- Commands: `orca-evaluate-idea`, `orca-validate-idea`, `orca-spec`
- Skills: `orca-business-ideation`
