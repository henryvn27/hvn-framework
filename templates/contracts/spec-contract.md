# Spec Contract

## Required Fields

- Goal
- Background
- Users And Scenarios
- Requirements
- Non-Goals
- Edge Cases
- Acceptance Criteria
- Assumptions
- Open Questions

## Optional Fields

- Linked issue or project
- Source citations
- Release or rollout notes

## Good Enough

- Requirements and acceptance criteria are specific enough for build and QA.
- Non-goals are explicit.
- Assumptions are separated from confirmed facts.
