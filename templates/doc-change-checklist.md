# Documentation Change Checklist

- Canonical detail page updated
- Relevant routing index updated
- `README.md` updated if newcomer experience changed
- Wiki updated if top-level browse experience changed
- Example added or updated if the behavior is non-obvious
- Ownership or freshness metadata reviewed
- `docs/recent-doc-updates.md` updated for meaningful doc changes
- `./scripts/docs-audit.sh` run successfully
- `./scripts/validate-repo.sh` run successfully
