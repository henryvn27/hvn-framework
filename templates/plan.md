# Plan

## Objective

State the spec or task this plan implements in one concrete sentence.

## Phases

For each phase, include:

- Phase name grounded in the real change
- Goal
- Expected files or systems touched
- Implementation notes
- Verification command or evidence
- Rollback or recovery consideration
- Phase result that should be reviewable when this phase is done

## Dependencies

List dependencies and sequencing constraints that materially affect execution order.

## Next Recommended Phase

State the single phase ORCA should execute first if the user approves implementation now.

## Review Gates

List code, design, security, and QA gates required before release.

## Status

Track each phase as pending, in progress, complete, or blocked.

## Anti-Generic Check

- Phase names should describe the actual work, not abstract project-management buckets.
- Verification should name the real command, screen, test, or evidence artifact.
