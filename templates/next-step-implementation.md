# Next Step After Implementation

## What Was Completed

Approved scope was implemented.

## Recommended Next Action

Run review and QA against the spec or milestone verification contract.

## Optional Alternate Action

Pause for approval if implementation changed scope, dependencies, architecture, or destructive behavior.

## What To Avoid

Do not call implementation done without verification evidence.

## Approval Or Review

Ask for review when behavior, maintainability, security, or UX risk is non-trivial.

## Done Means

The next phase is done when review and QA findings are resolved, accepted, or converted into follow-up work.
